# Feature: PWA

## Goal
- CONFIRMED: Register the app as a PWA with offline-ready assets and update prompts.

## Current Implemented Behavior
- CONFIRMED: VitePWA config defines app manifest name/short name `Balancer`, white theme color, and 192/512 icons.
- CONFIRMED: VitePWA includes favicon and PrimeIcons assets and uses Workbox cleanup/outdated cache behavior.
- CONFIRMED: Dev PWA support is enabled during `vite serve` with module service worker and `navigateFallback: "index.html"`.
- CONFIRMED: `src/helpers/pwa.ts` calls `useRegisterSW({ immediate: true, ... })`.
- CONFIRMED: Because `reload` is true in `vite.config.ts`, `__RELOAD_SW__` is replaced with `"true"` and registered service workers are checked every 120 seconds.
- CONFIRMED: `App.vue` displays a bottom-center PWA update toast when `needRefresh` becomes true.
- CONFIRMED: Clicking "Update now" removes the toast and calls `updateServiceWorker(true)`.

## User Flows
- CONFIRMED: App loads and registers service worker immediately.
- CONFIRMED: User sees an update-available toast when a new service worker needs refresh.
- CONFIRMED: User clicks update now to activate/reload through the PWA register helper.

## Inputs And Outputs
- CONFIRMED: Inputs are generated service-worker registration callbacks and `needRefresh` ref.
- CONFIRMED: Outputs are browser service-worker registration, cached build assets, and update toast UI.

## Data Files Read/Written
- CONFIRMED: Writes no finance JSON files.
- CONFIRMED: Uses browser service worker/cache storage generated by VitePWA.

## Store / Helper / Component Files Involved
- CONFIRMED: `vite.config.ts`, `src/helpers/pwa.ts`, `src/App.vue`, `src/claims-sw.ts`, `src/prompt-sw.ts`, public PWA icon files.

## Error Handling
- CONFIRMED: PWA callbacks log registration/offline/refresh events to console.
- UNCLEAR: Registration failures are not explicitly handled in app code.

## Edge Cases
- CONFIRMED: Source service-worker variants exist, but active runtime behavior depends on VitePWA generation strategy/config.
- CONFIRMED: Manifest icon paths intentionally mix slash/no-slash paths, with comments indicating test sensitivity.
- INFERRED: Offline app shell does not guarantee offline availability of remote JSON storage data unless already cached in IndexedDB.

## Acceptance Criteria
- CONFIRMED: GIVEN `initPWA()` runs, WHEN `useRegisterSW()` is called, THEN it is called with immediate registration.
- CONFIRMED: GIVEN `needRefresh` becomes true, WHEN `App.vue` observes it, THEN a persistent update toast is displayed.
- CONFIRMED: GIVEN the update toast is visible, WHEN the user clicks "Update now", THEN the toast is removed and `updateServiceWorker(true)` is invoked.
- CONFIRMED: GIVEN the app is built with VitePWA config, WHEN the manifest is generated, THEN it contains app name/short name `Balancer` and configured 192/512 icons.
- UNCLEAR: The expected user-facing behavior for service-worker registration failure is not specified.
